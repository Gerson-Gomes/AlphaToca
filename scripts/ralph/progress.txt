- **Learnings for future iterations:**
  - The `vi.mock(import('@prisma/client'), async (importOriginal) => ...)` syntax is from vitest's typed-mock API — passing a literal `import(...)` expression (not a string) gives `importOriginal` a proper return type so the spread `{ ...actual, PrismaClient: ... }` preserves the enum exports without a `any` cast. Required the `as unknown as typeof actual.PrismaClient` tail-cast on the replacement class because a test double doesn't match the full PrismaClient type signature.
  - When `prisma/seed.ts` imports the Prisma singleton via `import prisma from '../src/config/db'`, the mock on `@prisma/client` transparently intercepts the `new PrismaClient()` call inside `src/config/db.ts` — no need to separately mock `src/config/db`. `vi.hoisted` ensures the `mockPrismaClient` reference resolves inside the `vi.mock` factory.
  - `mock.calls[0][0]` is the first positional arg of the first call. For `prisma.user.createMany({ data: [...] })`, it's the `{ data }` object, so the payload is `mock.calls[0][0].data`. Cast to a concrete record shape to satisfy tsc without `any`.
  - `prisma/generators.ts` uses `faker.string.uuid()` which emits RFC 4122 v4. That's why the UUID assertion accepts both the fixed demo UUIDs and any v4-shaped id — random seed data already satisfies the strict regex.
  - When the mock for `@prisma/client` is missing a table used by `prisma/seed.ts` (e.g. `propertyImage`, `rentalDocument`), the missing call becomes a silent `undefined is not a function` at runtime IF `createMany` is called on it. Keep the mock in sync with `prisma/seed.ts` — whenever seed.ts adds a new table, mirror it in the mock.
